    Add a permission level to a Group in regards to a Portfolio.

    addPortfolioPermissionsForGroup(int $portfolio, int $group, int $permission) : bool

    Appends a specified permission level to the Group's current list of permissions. The appended permission cascades down the Portfolio tree (i.e., the Group will have permission for all sub-Portfolios as well).

    Parameters

    $portfolio

    int

    The identifier of the Portfolio the Group is being assigned to.

    $group

    int

    The identifier of the Group object recieving permissions for the Portfolio.

    $permission

    int

    The type of permission being granted to the Group object, as specified in 'constants.php'.

    Returns

    boolTrue if successful, false otherwise.

    Retrieve a number of Portfolio objects the current User is included in.

    getIncludedPortfolios(int $count, int $order_by, int $pos) : array | bool

    Retrieves a specified number of Portfolio objects. A User is 'included in' a Portfolio if they have any Projects they are a 'member' of included in the Portfolio or its sub-Portfolios.

    Parameters

    $count

    int

    The number of Portfolio objects desired (0 = return all such Portfolios).

    $order_by

    int

    The field that the Portfolios should be ordered by (ex: date_descending), as specified in 'constants.php'.

    $pos

    int

    The index of the first Portfolio to return in the set 'order_by' specifies.

    Returns

    arrayboolAn array of Portfolio objects if successful, false otherwise.
